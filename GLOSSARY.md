# GLOSSARY: Output File Documentation

This document provides comprehensive definitions for all columns and fields in the output files generated by the `asleep` package.

## Overview

The `asleep` package generates the following output files:
- **predictions.csv**: Epoch-by-epoch sleep predictions  
- **sleep_block.csv**: Sleep start/end times for each sleep period
- **day_summary.csv**: Daily sleep statistics and hourly breakdowns
- **summary.json**: Aggregated summary statistics across days

---

## predictions.csv

Contains 30-second epoch predictions for the entire recording period.

| Column | Type | Description |
|--------|------|-------------|
| `time` | timestamp | Exact timestamp for each 30-second epoch (YYYY-MM-DD HH:MM:SS.mmm) |
| `sleep_wake` | string | Binary sleep classification: `'sleep'`, `'wake'`, or `'non_wear'` |
| `sleep_stage` | string | Detailed sleep stage: `'wake'`, `'NREM'`, `'REM'`, or `'non_wear'` |
| `raw_label` | integer | Raw model output: `-1` (non-wear), `0` (wake), `1-3` (NREM stages), `4` (REM) |
| `temperature` | float | Mean temperature for epoch (°C) - only if `--report_light_and_temp` used |
| `light` | float | Mean light level for epoch (lux) - only if `--report_light_and_temp` used |

### Label Mappings
- **raw_label → sleep_wake**: `-1`→`'non_wear'`, `0`→`'wake'`, `1-4`→`'sleep'`
- **raw_label → sleep_stage**: `-1`→`'non_wear'`, `0`→`'wake'`, `1-3`→`'NREM'`, `4`→`'REM'`

---

## sleep_block.csv

Contains information about detected sleep periods after block validation and gap filling.

| Column | Type | Description |
|--------|------|-------------|
| `start` | timestamp | Sleep start time |
| `end` | timestamp | Sleep end time |
| `interval_start` | timestamp | Start of 24-hour analysis interval (12:00 noon) |
| `interval_end` | timestamp | End of 24-hour analysis interval (11:59:59 next day) |
| `wear_duration_H` | float | Device wear time (in hours) in the 24-hour analysis interval |
| `is_longest_block` | boolean | `True` if this is the longest/primary sleep period in the 24-hour analysis interval|

### Notes
- Sleep blocks must be ≥30 minutes duration to be included
- Gaps ≤60 minutes between valid sleep blocks are automatically filled
- Each row represents one continuous sleep period
- Primary sleep analysis focuses on `is_longest_block=True` entries

---

## day_summary.csv

Daily sleep statistics computed for the longest sleep block per 24-hour period.

### Basic Information
| Column | Type | Description |
|--------|------|-------------|
| `start_day` | string  Start date of the 24-hour analysis interval (YYYY-MM-DD format) |
| `day_of_week` | integer | Day of week: 0=Monday, 1=Tuesday, ..., 6=Sunday |
| `wear_duration_H` | float | Device wear time (in hours) in the 24-hour analysis interval |
| `is_weekend` | boolean | `True` for Friday and Saturday (days before non-work days) |

### Core Sleep Metrics
| Column | Type | Description |
|--------|------|-------------|
| `sol_min` | float | **Sleep Onset Latency**: Time from bedtime to sleep onset (minutes) |
| `tst_min` | float | **Total Sleep Time**: Duration of all sleep stages combined (minutes) |
| `waso_min` | float | **Wake After Sleep Onset**: Wake periods after initial sleep onset (minutes) |
| `reml_min` | float | **REM Latency**: Time from sleep onset to first REM epoch (minutes) |
| `se_perc` | float | **Sleep Efficiency**: TST/(time in bed) as proportion (0.0-1.0) |

### Sleep Stage Durations (minutes)
| Column | Type | Description |
|--------|------|-------------|
| `wake_min` | float | Total wake time during sleep period |
| `n1_min` | float | NREM Stage 1 (light sleep) duration |
| `n2_min` | float | NREM Stage 2 (light sleep) duration |
| `n3_min` | float | NREM Stage 3 (deep sleep) duration |
| `nrem_min` | float | Total NREM sleep (N1+N2+N3) duration |
| `rem_min` | float | REM sleep duration |

### Hourly Breakdown Columns
Pattern: `{hour}_hour_{stage}_min` where:
- **hour**: 0-23 (hour of day)
- **stage**: `wake`, `n1`, `n2`, `n3`, `nrem`, `rem`, `tst`

Examples:
- `0_hour_wake_min`: Wake time during midnight hour (00:00-00:59)
- `1_hour_tst_min`: Total sleep time during 1 AM hour (01:00-01:59)
- `23_hour_rem_min`: REM sleep during 11 PM hour (23:00-23:59)

## summary.json

Aggregated statistics across all valid days (≥22 hours wear time by default).

### Field Naming Convention
Format: `{timeperiod}_{statistic}_{metric}`

- **timeperiod**: `overall`, `monday`, `tuesday`, ..., `sunday`, `weekday`, `weekend`
- **statistic**: `mean`, `median`
- **metric**: Any column from day_summary.csv or hourly breakdowns

### Overall Statistics
| Field Pattern | Description |
|---------------|-------------|
| `overall_mean_{metric}` | Mean value across all valid days |
| `overall_median_{metric}` | Median value across all valid days |

### Day-of-Week Statistics
| Field Pattern | Description |
|---------------|-------------|
| `monday_mean_{metric}` | Mean for Mondays only |
| `tuesday_mean_{metric}` | Mean for Tuesdays only |
| ... | (continues for all days) |

### Weekend/Weekday Statistics
| Field Pattern | Description |
|---------------|-------------|
| `weekday_mean_{metric}` | Mean for weekdays (Mon-Fri) |
| `weekend_mean_{metric}` | Mean for weekends (Fri-Sat) |

### Validity Counts
| Field | Description |
|-------|-------------|
| `num_valid_days` | Total days meeting minimum wear time requirement |
| `num_valid_weekday` | Valid weekdays (Mon-Fri) |
| `num_valid_weekend` | Valid weekend days (Fri-Sat) |

### Example Fields
```json
{
  "overall_mean_tst_min": 433.42,
  "overall_median_se_perc": 0.94,
  "monday_mean_sol_min": 15.5,
  "weekend_mean_rem_min": 8.2,
  "overall_mean_2_hour_wake_min": 5.5,
  "num_valid_days": 6
}
```

## Sleep Stage Definitions

### Raw Label Mappings (from macros.py)
- **-1**: Non-wear period (device not being worn)
- **0**: Wake state
- **1**: NREM Stage 1 (light sleep, sleep initiation)
- **2**: NREM Stage 2 (light sleep, sleep spindles/K-complexes)  
- **3**: NREM Stage 3 (deep sleep, slow-wave sleep)
- **4**: REM sleep (rapid eye movement, dreaming)

### Clinical Sleep Metrics

#### Sleep Onset Latency (SOL)
- Time from bedtime to sustained sleep (≥1.5 minutes of non-wake stages)
- Calculated using 3-epoch threshold (sleep_stats.py:46-55)

#### Sleep Efficiency (SE)
- Proportion of time in bed actually spent sleeping
- Formula: Total Sleep Time / Time in Bed
- Healthy adults typically achieve >85% efficiency

#### Wake After Sleep Onset (WASO)  
- Total wake time occurring after initial sleep onset
- Excludes initial wake time before sleep begins
- Key indicator of sleep maintenance quality

#### REM Latency (REML)
- Time from sleep onset to first REM epoch
- Normal range: 70-120 minutes in healthy adults
- Calculated as: First REM time - Sleep Onset Latency

## Analysis Windows

### Day Boundaries
- **24-hour intervals**: 12:00 noon to 11:59:59 next day

### Minimum Wear Time
- Default: 22 hours per 24-hour analysis interval (configurable with `--min_wear`)
- Intervals below threshold excluded from summary statistics

### Sleep Periods
- **Block validation**: At least 30 minutes continuous sleep
- **Gap filling**: Bridges ≤60 minute interruptions between valid blocks
- **Primary sleep**: Longest period per interval used for daily statistics

---

## File Relationships

1. **Raw processing**: Accelerometer data → predictions.csv (30-sec epochs)
2. **Block detection**: predictions.csv → sleep_block.csv (sleep periods)  
3. **Daily analysis**: sleep_block.csv → day_summary.csv (per-day metrics)
4. **Aggregation**: day_summary.csv → summary.json (multi-day statistics)

Each file provides different temporal resolution and analytical focus for comprehensive sleep assessment.
